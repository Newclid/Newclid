JGEX definitions
================

Definitions are the basic building blocks of JGEX problems.
Each definition works as a function, demanding a certain collection of arguments in order to create new points, and adding corresponding predicates to the proof state.

Some definitions can be used in combination with others, corresponding to the intersection of constructions.
That is stated in the clause defining a point by two definitions separated by ",".

Here in the documentation you can recognize the definitions that can be used in intersections because the "Construction" field returns an object that is neither a point, nor a group of points.

The definitions available are listed in the code in the ``JGEXConstructionName`` enumeration (see :doc:`../../../../source/newclid.jgex.constructions`).

The definitions in section :ref:`New Definitions` below were added by us, the other ones were present in the original AlphaGeometry codebase.

The original AlphaGeometry also had definitions "on_aline2" and "cc_tangent0", which were not functional, and "on_opline", which depended on the half-line class, which was eliminated. We also renamed the previous "eq_trapezoid" definition to "iso_trapezoid".
